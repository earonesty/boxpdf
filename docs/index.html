Declarative PDFs
that run anywhere.

A tiny box-layout DSL over pdf-lib. Flexbox-lite for server-side PDF generation in Node, Cloudflare Workers, Deno, and the browser. No WASM. No headless browser. No React tax.

import { cleanTheme, hline, hstack, renderFlow, text, vstack } from "boxpdf";
import { PDFDocument, StandardFonts } from "pdf-lib";

const pdf = await PDFDocument.create();
const font = await pdf.embedFont(StandardFonts.Helvetica);
const bold = await pdf.embedFont(StandardFonts.HelveticaBold);
const theme = cleanTheme(font, bold);

await renderFlow(pdf, [
  vstack({ gap: 8 },
    text("Receipt #18472", theme.type.h1),
    text("May 14, 2026",     theme.type.caption)
  ),
  hline(theme.hr),
  hstack({ gap: 16 },
    text("Wool socks",   theme.type.body),
    text("$28.00",        { ...theme.type.body, font: bold, align: "right", width: 80 })
  )
]);

const bytes = await pdf.save(); // -> Uint8Array

What's in the box

Runs at the edge

The only declarative PDF lib that works on Cloudflare Workers today. No WASM, no fontkit-trapped-in-browser, no  nodejs_compat.

Flex-ish layout, no coordinates

vstack / hstack with padding, margin, gap, justify, align, grow, real word-wrapping, ellipsis. No page.drawText(x, height − 246).

Themes you can recognize

Drop-in cleanTheme, stripeTheme, editorialTheme, brutalistTheme. Same code, totally different aesthetic.

Multi-page primitive

renderFlow paginates with atomic children. keepTogether bundles widows. Page headers / footers with { pageNumber, totalPages } built in.

Pull in any font

Optional boxpdf/inter ships subsetted Inter (82 KB / weight). Or loadFont(pdf, source) + boxpdf font add to bundle any TTF you own.

Hyperlinks, decorations, metadata

link({ href }, ...), underline / strikethrough, title / author / subject as renderFlow options.

Debug overlay

renderFlow(pdf, nodes, { debug: true }) outlines every content + margin box in red and orange. Trace layouts visually.

Tiny, treeshakable

Core is <7 KB minified. boxpdf/inter and @pdf-lib/fontkit only load when you use them. Real bundle hygiene.

Four named themes

Same identical layout code. Drop in a different theme; the document transforms.

Template gallery

Production-ready documents in templates/. Copy a file, edit it for your data, ship it.

Install

npm install boxpdf pdf-lib

pdf-lib is a peer dependency. If you want custom-font embedding (incl. boxpdf/inter), @pdf-lib/fontkit comes along for the ride and is lazy-loaded only when you embed a non-standard font.

Cloudflare Workers

Both the core and the boxpdf/inter subpath are verified to run on Cloudflare Workers without nodejs_compat or any WASM flag. Drop it into a worker handler:

import { Hono } from "hono";
import { PDFDocument, StandardFonts } from "pdf-lib";
import { cleanTheme, renderFlow, text, vstack } from "boxpdf";

const app = new Hono();

app.get("/receipt.pdf", async (c) => {
  const pdf  = await PDFDocument.create();
  const font = await pdf.embedFont(StandardFonts.Helvetica);
  const bold = await pdf.embedFont(StandardFonts.HelveticaBold);
  const t    = cleanTheme(font, bold);
  await renderFlow(pdf, [
    text("Thanks!", t.type.h1),
    text("This PDF was generated at the edge.", t.type.body)
  ]);
  const bytes = await pdf.save();
  return new Response(bytes, { headers: { "content-type": "application/pdf" } });
});

export default app;

Sizes are rough; measure yourself before believing any table. boxpdf is the right pick when you want declarative layout AND your runtime is anything other than long-lived Node. pdf-lib is still the right pick if you want raw coordinate control and don't care about layout primitives. @react-pdf/renderer is the right pick if you're already deep in React on Node and don't need Workers.
